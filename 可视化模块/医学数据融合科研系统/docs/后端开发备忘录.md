CREATE TABLE visit_records (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,

  patient_id VARCHAR(18) NOT NULL,

### 3. 不同账户信息隔离

**需求说明：**
- 不同用户只能查看自己创建的随访记录
- 管理员可以查看所有记录
- 数据权限隔离

**技术要求：**
- 实现基于角色的访问控制（RBAC）
- 数据权限过滤
- 接口权限验证

**角色权限设计：**
- **管理员（admin）**：查看所有随访记录，管理用户
- **医生（doctor）**：查看自己创建的随访记录
- **护士（nurse）**：查看自己创建的随访记录

**权限过滤规则：**
```typescript
// 获取随访记录时的权限过滤
function getVisitRecords(userId: string, userRole: string) {
  if (userRole === 'admin') {
    // 管理员查看所有记录
    return findAllVisits()
  } else {
    // 普通用户只看自己的记录
    return findVisitsByCreatorId(userId)
  }
}
```

**数据库查询优化：**
```sql
-- 普通用户查询
SELECT * FROM visit_records WHERE creator_id = ? ORDER BY create_time DESC;

-- 管理员查询
SELECT * FROM visit_records ORDER BY create_time DESC;
```

**中间件实现：**
```typescript
// 权限验证中间件
function checkPermission(requiredRole: string) {
  return (req, res, next) => {
    const user = req.user;
    if (user.role === 'admin' || user.role === requiredRole) {
      next();
    } else {
      res.status(403).json({ error: '无权访问' });
    }
  };
}
```
